## 2. Top 5 Most Critical Strategic Assets

The following five individual assets represent the highest concentration of security risk and operational dependency across the MedDefense enterprise footprint.

### 1. `ehr-db-01` (Core EHR Database Server)
* **Strategic Evaluation:** This database server is the ultimate "crown jewel" of MedDefense Health Systems. It stores the primary, live database of Protected Health Information (PHI) for every active patient, including diagnoses, allergies, historical treatments, and active prescriptions. A failure in $Confidentiality$ triggers a mandatory federal investigation, massive civil litigation, and millions in statutory HIPAA fines. A failure in data $Integrity$ or $Availability$ immediately threatens patient lives, as clinical staff lose real-time access to critical care vectors, rendering them unable to safely verify cross-contamination risks or medication interactions.

### 2. `FW-01` (FortiGate 100F Perimeter Firewall)
* **Strategic Evaluation:** This security device serves as the single gatekeeper protecting MedDefense’s internal environment from the public internet, while simultaneously routing the site-to-site VPN connectivity that links the Westside Clinic and Corporate HQ to Central Hospital. It enforces the boundary architecture that shields vulnerable, unpatchable legacy endpoints like the Windows XP MRI terminal. If this perimeter firewall is compromised or suffers an availability failure, the outer defensive line collapses entirely. Attackers gain unfettered network access to internal subnets, and remote clinics are immediately cut off from accessing core hospital applications.

### 3. `ehr-srv-01` (Core EHR Application Server)
* **Strategic Evaluation:** While `ehr-db-01` houses the raw records, this application server provides the user interface and processing logic that allows clinical staff to read, update, and interact with patient data. If this application server goes down, clinical workflows transition instantly to paper records, heavily reducing operational efficiency and increasing the risk of manual data entry errors. The previous 9-hour database migration failure proved that dropping this server severely disrupts clinical operations and places a heavy administrative burden on hospital staff.

### 4. `NAS-01` (Synology Network Attached Storage)
* **Strategic Evaluation:** This hardware asset acts as the sole target for all nightly virtual machine backups orchestrated by Veeam. Because it stores the compressed, duplicated image sets of all active servers, its data integrity and availability are paramount to corporate survival. The current configuration places this device in the same physical room and network subnet as the production infrastructure, creating a severe single point of failure. If an attacker leverages lateral movement to wipe or encrypt this asset alongside the main servers, MedDefense loses its primary recovery capability, turning an operational disruption into a permanent data loss crisis.

### 5. `pacs-srv-01` (PACS Imaging Engine Server)
* **Strategic Evaluation:** This server processes and delivers critical diagnostic imagery (MRIs, CT scans, and X-rays) directly to emergency departments and surgical wards. Modern specialized medicine cannot function without immediate, trusted visual diagnostics. If this engine is knocked offline, or if its data integrity is silently compromised (e.g., misassociating or corrupting surgical imaging files), physicians are forced to delay surgeries or proceed without precise anatomical telemetry. Furthermore, because this asset is explicitly excluded from nightly automated backups, any catastrophic failure on this machine results in the immediate, permanent destruction of historical medical imagery.
